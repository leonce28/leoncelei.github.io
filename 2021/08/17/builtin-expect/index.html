__builtin_expect指令

这个指令是gcc引入的，作用是允许程序员将最有可能执行的分支告诉编译器。这个指令的写法为：__builtin_expect(EXP, N)。
意思是：EXP == N的概率很大。

今天在撸代码的时候发现有个likely的宏函数，不知道是干嘛的，直接追踪到宏定义，发现如下代码：

于是，查了查__builtin_expect，发现这是个gcc引入的指令。两者的异同如下：

  • if (likely(value))和if (unlikely(value))都等价于if (value)。
  • likely(value)表示value大概率为真，小概率为假。
  • unlikely(value)表示value小概率为真，即大概率为假。

也就是说，使用likely()，执行 if 后面的语句的机会更大，使用 unlikely()，执行 else 后面的语句的机会更大。通过这种方式，编译器在编译过程中，会将可能性更大的代码紧跟着起面的代码，从而减少指令跳转带来的性能上的下降。
