clickhouse数据库是一个列数据库，在原来的工作生涯中从来没有接触过。刚好目前工作后续有这块的业务，而且自己也对列数据库比较感兴趣，那就以clickhouse作为我学习的第一个列数据库，梳理相关知识，提高自身竞争力。

clickhouse学习笔记

一、clickhouse安装

1.1 硬件环境

操作系统要求：

只要是Linux，64位都可以

Ubuntu：

优先支持Ubuntu，Ubuntu有官方编译好的安装包可以使用

Centos和RedHat：

其次是CentOS和RedHat，有第三方组织altinity编译好的rpm包可以使用

第三方altinity的下载地址：https://packagecloud.io/altinity/clickhouse

其他Linux系统：

如果是其他Linux系统，需要自己编译源码

具体查看官网：https://clickhouse.tech/docs/zh/getting_started/install/#from-sources

CPU要求：

机器的CPU必须支持SSE 4.2指令集

检测机器的cpu是否支持：

检测命令：grep -q sse4_2 /proc/cpuinfo && echo "SSE 4.2 supported" || echo "SSE 4.2 not supported"

1.2 安装步骤

  1. 创建目录

  2. 进入官网。https://packagecloud.io/altinity/clickhouse?filter=rpms

  3. 选择对应需要下载的文件。版本号要对应

  4. 按个执行下载命令

  5. 进入下载目录

  6. 执行rpm -ivh *.rpm命令进行安装即可。

1.3 服务验证

  1. 启动/停止服务，使用命令service clickhouse-server start/stop

  2. 启动服务

  3. 客户端尝试连接clickhouse server。本机直接用clickhouse-clien命令就可以进入。

1.4 简单使用

clickhouse-client的使用可以参考官方文档：https://clickhouse.tech/docs/zh/interfaces/cli/。

  1. 创建一张数据表。create table test( id UInt16, name String, create_date date) engine = MergeTree(create_date, (id), 8192)。

  2. 查看创建的表。默认情况下是default数据库。

  3. 插入三条测试数据并查询。insert into test values(1, 'tom', '2021-07-14'),(2,'汤姆', '2021-07-15'),(3,'\':)', '2021-07-16')

  4. 重复执行insert into语句，发现db中的数据又多了一块一样的。这个后续再研究，这个应该是clickhouse的特性。
